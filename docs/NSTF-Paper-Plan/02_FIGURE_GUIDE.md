### 1.3 Draw.io 绘制技巧

**颜色方案：**
- 背景: 白色
- M3-Agent 基础: 浅灰色 (#F5F5F5)
- Neural-Symbolic Memory: 浅蓝色边框 (#0366D6), 内部白色
- Memory Prototype: 橙色 (#FB8C00)
- Procedure DAG: 绿色 (#4CAF50)
- 箭头: 深灰色 (#333333)

**形状选择：**
- 模块: 圆角矩形 (Rounded Rectangle)
- 数据流: 箭头 (Arrow)
- DAG节点: 小圆形或小矩形
- 强调框: 虚线边框

**字体：**
- 标题: Arial Bold, 14pt
- 模块名: Arial, 12pt
- 说明文字: Arial, 10pt

## 2. Figure 2: E2P (Episodic-to-Procedural) 流程图

### 2.1 设计目标
展示如何从原始视频记忆中提取程序性知识

### 2.2 布局建议

```
┌─────────────────────────────────────────────────────────────────┐
│                    E2P Algorithm                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Input: Episodic Memories (clip 0-24)                          │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐     ┌─────┐                  │
│  │ e₀  │ │ e₁  │ │ e₂  │ │ e₃  │ ... │ e₂₄ │                  │
│  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘     └──┬──┘                  │
│     │       │       │       │           │                      │
│     └───────┴───────┴───────┴───────────┘                      │
│                      ↓                                          │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │             Step 1: Procedure Detection                  │   │
│  │  LLM analyzes memories to identify procedure patterns    │   │
│  │  Output: ["红烧肉制作", "摆盘装饰", ...]                   │   │
│  └──────────────────────────────────────────────────────────┘   │
│                      ↓                                          │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │             Step 2: Structure Extraction                 │   │
│  │  For each procedure, extract:                            │   │
│  │  - Steps (DAGNode)                                       │   │
│  │  - Dependencies (DAGEdge)                                │   │
│  │  - Temporal constraints                                  │   │
│  │  - Required resources                                    │   │
│  └──────────────────────────────────────────────────────────┘   │
│                      ↓                                          │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │             Step 3: Memory Prototype Creation            │   │
│  │  Aggregate embeddings from related clips:                │   │
│  │  prototype = Σ (w_i × e_i) / Σ w_i                       │   │
│  │  Dimension: 1536 = 512(text) + 512(visual) + 512(action) │   │
│  └──────────────────────────────────────────────────────────┘   │
│                      ↓                                          │
│  Output: ProcedureNode with DAG + Prototype                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  ┌────────────────┐      ┌─────────────────────────┐    │   │
│  │  │MemoryPrototype│  +   │   ProcedureDAG          │    │   │
│  │  │  [3072-dim]    │      │  steps, edges, alts...  │    │   │
│  │  └────────────────┘      └─────────────────────────┘    │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 Draw.io 技巧

**流程箭头样式：**
- 使用粗箭头 (3-4pt)
- 箭头颜色与步骤框匹配

**步骤框样式：**
- 每个 Step 使用不同的浅色背景
- Step 1: 浅蓝 (#E3F2FD)
- Step 2: 浅绿 (#E8F5E9)
- Step 3: 浅橙 (#FFF3E0)

## 6. Draw.io 导出设置

### 6.1 论文投稿格式

**导出为 PDF (矢量)：**
1. File → Export as → PDF
2. 选择 "Include a copy of my diagram"
3. Border: 0 或 5

**导出为 PNG (高清)：**
1. File → Export as → PNG
2. Zoom: 300% 或更高
3. Border: 10px
4. Background: Transparent 或 White

### 6.2 尺寸建议

- 单栏图: 宽度 3.5 inches (约 8.5 cm)
- 双栏图: 宽度 7 inches (约 17.5 cm)
- 高度: 根据内容调整，但不超过 9 inches
